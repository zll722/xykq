# 微信小程序端开发方案（三角色版）

## 一、现状分析

- 后端 API 已完整实现（50+ 接口），支持 USER/TEACHER/COUNSELOR/ADMIN 四种角色
- 微信开发者工具已安装，`project.config.json` 已存在（appid: `wx2c6c8ca122440e03`）
- 当前报错原因：项目根目录下没有 `app.json`，需要创建小程序工程并指定 `miniprogramRoot`
- 后端已有的角色 API：
  - **学生（USER）**：签到、课表、考勤记录、请假申请、消息、个人信息
  - **教师（TEACHER）**：工作台统计、我的班级、班级考勤报表、请假通知
  - **辅导员（COUNSELOR）**：工作台统计、班级管理、学生列表、请假审批（待审/历史）

## 二、技术方案

| 项目 | 选择 | 理由 |
|------|------|------|
| 框架 | 原生微信小程序 | 无需额外构建工具，开发者工具直接预览，毕设更能体现技术能力 |
| 多角色方案 | 自定义 tabBar + 登录后角色分发 | 不同角色展示不同 tabBar 和页面 |
| 网络请求 | 封装 wx.request + JWT Bearer Token | 复用现有后端认证，零改动 |
| 状态管理 | app.globalData + 页面 data | 小程序原生方案，简单够用 |

## 四、三角色功能规划

### 4.1 学生端（USER）

**TabBar：首页 | 课程 | 请假 | 我的**

| 页面 | 对应 API | 功能 |
|------|----------|------|
| 首页 | `GET /stats/my-overview`、`GET /schedules/my` | 出勤率统计、今日课程、快捷签到 |
| 课程表 | `GET /schedules/my` | 按周展示课程安排 |
| 签到 | `POST /attendance/sign-in` | 课程签到（可集成定位） |
| 考勤记录 | `GET /attendance/records/my`、`GET /stats/my-calendar` | 历史考勤、日历视图 |
| 请假申请 | `POST /leave/requests` | 类型/原因/时间/附件 |
| 请假历史 | `GET /leave/requests/my` | 历史记录列表 |
| 审批进度 | `GET /leave/requests/my-progress` | 实时状态 |
| 消息 | `GET /messages`、`PUT /messages/{id}/read` | 通知列表、标记已读 |
| 个人中心 | `GET /users/me`、`PUT /users/me`、`POST /auth/change-password` | 信息展示/修改/改密/退出 |

### 4.2 教师端（TEACHER）

**TabBar：工作台 | 考勤 | 请假 | 我的**

| 页面 | 对应 API | 功能 |
|------|----------|------|
| 工作台 | `GET /teacher/dashboard` | 今日课程数、学生总数、出勤率概览 |
| 我的班级 | `GET /teacher/classes` | 授课班级列表 |
| 考勤报表 | `GET /teacher/classes/{classId}/attendance` | 按班级查看考勤详情 |
| 请假通知 | `GET /teacher/leave-notifications` | 学生请假通知列表 |
| 个人中心 | `GET /users/me`、`POST /auth/change-password` | 信息/改密/退出 |

### 4.3 辅导员端（COUNSELOR）

**TabBar：工作台 | 班级 | 审批 | 我的**

| 页面 | 对应 API | 功能 |
|------|----------|------|
| 工作台 | `GET /counselor/dashboard` | 待审批数、班级数、学生数概览 |
| 班级管理 | `GET /counselor/classes` | 管辖班级列表 |
| 学生列表 | `GET /counselor/classes/{classId}/students` | 班级学生详情 |
| 待审批 | `GET /counselor/leave/pending` | 待审批请假列表 |
| 审批操作 | `POST /counselor/leave/{requestId}/approve` | 通过/驳回 + 意见 |
| 审批历史 | `GET /counselor/leave/history` | 已处理记录 |
| 个人中心 | `GET /users/me`、`POST /auth/change-password` | 信息/改密/退出 |

## 五、多角色路由方案

小程序不支持动态路由，采用以下方案：

1. **所有页面都注册在 `app.json` 的 `pages` 中**（不使用原生 tabBar）
2. **使用自定义 tabBar 组件**：登录后根据 `role_code` 渲染不同的 tab 项
3. **登录后跳转逻辑**：
   - `USER` → `/pages/student/index/index`
   - `TEACHER` → `/pages/teacher/index/index`
   - `COUNSELOR` → `/pages/counselor/index/index`
4. **页面级权限守卫**：每个页面 `onLoad` 时检查角色，不匹配则跳回登录

## 六、后端适配

**后端 API 零改动**，小程序直接复用现有接口：
- 认证：`POST /api/v1/auth/login` 获取 JWT token
- 请求头：`Authorization: Bearer <token>`
- 文件上传：`wx.uploadFile` 对接 `POST /api/v1/files/upload`
- 开发阶段：微信开发者工具勾选「不校验合法域名」，请求 `http://localhost:8080`

## 七、实现步骤

### 步骤 1：工程初始化
- 修改 `project.config.json` 添加 `"miniprogramRoot": "miniprogram/"`
- 创建 `miniprogram/app.js`、`app.json`、`app.wxss`
- `app.json` 注册所有页面路径（不配置原生 tabBar）
- 全局样式定义（主色 #409EFF、字体、间距）

### 步骤 2：工具层与 API 层
- `utils/request.js`：Promise 封装 wx.request，自动注入 token，统一错误处理
- `utils/auth.js`：token/userInfo 存取，角色判断，登录态检查
- `api/` 下各模块文件，对接后端已有接口

### 步骤 3：自定义 tabBar 组件
- `components/custom-tabbar/`：接收 `role` 属性，渲染对应 tab 列表
- 学生 4 tab / 教师 4 tab / 辅导员 4 tab
- 处理 tab 切换跳转（`wx.reLaunch`）

### 步骤 4：登录页
- 账号密码表单
- 登录成功后存储 token + userInfo（含 role_code）
- 根据角色跳转对应首页

### 步骤 5：学生端页面（7 页）
- 首页（统计卡片 + 今日课程 + 签到入口）
- 课程表（周视图）
- 签到页
- 考勤记录（列表 + 筛选）
- 请假申请（表单 + 文件上传）
- 请假历史 / 审批进度
- 消息通知
- 个人中心

### 步骤 6：教师端页面（4 页）
- 工作台（统计概览）
- 我的班级 + 考勤报表
- 请假通知
- 个人中心

### 步骤 7：辅导员端页面（5 页）
- 工作台（统计概览）
- 班级管理 + 学生列表
- 待审批 + 审批操作
- 审批历史
- 个人中心

### 步骤 8：验证与调试
- 确保微信开发者工具编译通过
- 三种角色登录后分别进入对应界面
- 核心流程联调（签到、请假、审批）
